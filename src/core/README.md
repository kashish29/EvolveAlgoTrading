# Core Module

This module defines the fundamental data structures and enumerations used throughout the Algorithmic Trading Framework.

## `enums.py`
Contains enumerations for standardized types such as:
- `OrderType`: (MARKET, LIMIT, etc.)
- `OrderStatus`: (PENDING, FILLED, CANCELLED, etc.)
- `TradeType`: (BUY, SELL)
- `InstrumentType`: (EQUITY, FUTURE, etc.)
- `Timeframe`: (1MIN, 5MIN, 1D, etc.)

## `models.py`
Contains data classes for core entities:
- `Candle`: Represents a single price bar (OHLCV).
- `Signal`: Represents a trading signal generated by a strategy.
- `Order`: Represents a request to the broker to execute a trade.
- `Trade`: Represents an executed trade (a fill or partial fill of an order).
- `Position`: Represents the net holdings of a particular instrument.

These models are designed to be simple, efficient, and provide a clear structure for data transfer between different components of the framework.
