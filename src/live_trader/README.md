# Live Trader Module (`src/live_trader/`)

This module is responsible for simulating the execution of trading strategies. It handles the flow of simulated real-time market data, processes signals from strategies, converts signals to orders, and simulates order placement with a mock broker.

## Key Components:

### `event_handler.py`
-   **`DataFeedSimulator`**:
    -   Simulates the generation of market data (bars/candles) from historical data.
    -   Simulates order updates (fills, rejections) by interacting with the `MockFyersClient`'s order processing logic.
    -   Yields both market data events and order update events.
-   **`EventHandler`**:
    -   Drives the simulation loop.
    -   Consumes events from the `DataFeedSimulator`.
    -   Dispatches market data events (`Candle`) to the trading strategy's `on_bar` method.
    -   Dispatches order update events (`Order`) to the trading strategy's `on_order_update` method.

### `signal_processor.py`
-   **`SignalProcessor`**:
    -   Receives `Signal` objects generated by active trading strategies.
    -   (Future Integration) Performs pre-trade risk checks using a `RiskManager`. Currently, this step is a placeholder.
    -   If risk checks pass (or no risk manager is present), forwards the `Signal` to the `ExecutionHandler`.

### `execution_handler.py`
-   **`ExecutionHandler`**:
    -   Receives `Signal` objects from the `SignalProcessor`.
    -   Converts `Signal` objects into `Order` objects suitable for the broker.
    -   Places these `Order` objects using the `broker_client` (e.g., `MockFyersClient`).
    -   Logs the order placement attempt and the broker's response.

## Workflow:

1.  The **`DataFeedSimulator`** generates a market data event (new bar).
2.  The **`EventHandler`** receives the market data and passes it to the strategy's `on_bar` method.
3.  The strategy, based on its logic, may generate a **`Signal`**.
4.  The strategy (or a component it calls) sends the `Signal` to the **`SignalProcessor`**.
5.  The **`SignalProcessor`** (optionally) validates the trade against risk rules.
6.  If approved, the `SignalProcessor` passes the `Signal` to the **`ExecutionHandler`**.
7.  The **`ExecutionHandler`** converts the `Signal` to an `Order` and sends it to the `broker_client` (e.g., `MockFyersClient`).
8.  The `MockFyersClient` simulates placing the order. If it's a market order, it might fill immediately. If it's a limit/stop order, it's added to open orders.
9.  The `DataFeedSimulator`, after yielding a market event, also checks the `MockFyersClient` for any order updates (e.g., fills of pending limit/stop orders based on the latest bar).
10. If there are order updates, the `DataFeedSimulator` yields them.
11. The **`EventHandler`** receives these order updates and passes them to the strategy's `on_order_update` method.
12. The loop continues with the next market data event.

This setup allows for a simulated live trading environment where strategy decisions, order placements, and market movements are processed sequentially.
