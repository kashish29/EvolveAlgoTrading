# Live Trader Module (`src/live_trader/`)

This module is responsible for the execution of trading strategies in a live market environment. It will handle real-time market data, process signals from strategies, manage order lifecycle with the broker, and continuously monitor trading activity.

**Note: The components in this module are currently placeholders.** They define the intended structure but do not contain functional implementations yet.

## Key Components (Future Implementation):

### `event_handler.py`
- **`EventHandler`:**
  - Will connect to the broker's WebSocket or API for real-time market data (ticks, candles).
  - Will receive order updates (fills, cancellations, rejections) from the broker.
  - Will dispatch market data to relevant strategies and order updates to the `ExecutionHandler` or a portfolio manager.

### `signal_processor.py`
- **`SignalProcessor`:**
  - Receives trading `Signal` objects generated by active strategies.
  - May perform pre-trade checks by consulting the `RiskManagement` module (e.g., position sizing, drawdown limits).
  - If checks pass, forwards the signal or a derived order request to the `ExecutionHandler`.

### `execution_handler.py`
- **`ExecutionHandler`:**
  - Manages the lifecycle of orders sent to the broker.
  - Translates `Signal` objects or internal order requests into broker-specific order parameters.
  - Places orders using the `BrokerAPI` client (e.g., `FyersClient`).
  - Tracks order statuses (pending, open, filled, cancelled, rejected).
  - Updates internal portfolio or position state based on fills.
  - May implement logic for order modification or cancellation.

---
Developing a robust live trading module requires careful attention to error handling, connectivity management, latency, and synchronization with the broker.
